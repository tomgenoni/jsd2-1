Geolocation Lab

    Exercise: Geolocation API

    Instructions

    The Geolocation API is what allows a web page to know where you are. In this intro to geolocation, we will get the latitude & longitude of your device and print it on the page. In future exercises you will build on this code to display your location on a map.

    Remember, a position JSON object is passed to your callback function from the Geolocation API. You'll have to unpack the JSON data to find the latitude & longitude properties.

    Tasks

    1. Update the event handler for the button to call getCurrentPosition()
      • When the button is clicked, the getCurrentPosition() function should be run to start a request for the location.
      • Now, remember, getCurrentPosition() needs to be told what its callback function is. So that everyone names the callback the same, please create a function named geoSuccess. This will be your callback function.
    2. Write the content of the geoSuccess callback function.
      • This is the function that will be called when the location is determined.
      • A Geolocation callback is always passed one argument, which is a Position object.
      • So that everyone names the callback variable the same, please name the one argument position.
      • At this point you must do some debugging to thoroughly understand the json structure of the Position object. You must find the latitude and longitude properties in the position object. And save those two into variables.
    3. Write a separate function that will be responsible for printing out the latitude and longitude on the screen.
      • This function should be called and passed two arguments: the latitude and longitude variables you just created in the previous step.
      • Technically, the DOM work should be achieved by adding a new <li> element to the list and setting the text of the <li> to be the lat/long.
      • For example:
        <li>37.76341336916038, -122.41807504195602</li>
      • Note: as part of the starter code, the parent list Element has already been stored in a variable named locations.

    Tips

    It *may* be helpful to do the last step first. Step 3 is just a "give me some data and I'll create some DOM Elements" kinda function. This is something we've already done and you can fake it and hardcode the two values for latitude and longitude to test it works first. Even by just calling it at the console and passing two values.

    This might prove to be a rewarding chunk to knock out first.
