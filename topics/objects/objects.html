Object syntax

  • Each property is separated by a colon, then the value.
  • Properties are separated by a comma
  • Each property name (aka key) is a string
  • Each property value can be any type of object. EG: a string, number, array, object

Notes

Objects can be categorized into three general buckets:

  1. Native JavaScript language objects

    These are objects that are part of the programming language itself. They would be the same whether you were running JavaScript in the client (browser) or on the server (node.js). These objects also that have similar counterparts in other programming languages (like swift, ruby, python, etc) Examples:

    • Array
    • Math
    • JSON

  2. Host Environment objects

    These host objects are specific to the environment JavaScript is running in. Since we're running JavaScript in the browser, these are things like:

    • Navigator
    • Window
    • Document (aka DOM)
    • Element
    • Event

  3. User/data objects

    The last category of objects are the ones you create. Whereas Native & Host objects are types of things pre-defined by standards commitees, these are things you define yourself. Most commonly, these are to model data in real life. You pick the name, you pick the data model structure. Some examples:

    • reservation
    • business
    • contact
    • task
