JSD2: Topics: Handlebars

Handlebars

Handlebars is a template engine. It combines a template with a data source to create HTML.

For Javascript template systems, like Handlebars, that template is a string of HTML and the data source is a JSON object. The output is also a string of HTML, with the data properties from the JSON merged in.

Objectives

You should be comfortable with:

  1. Writing a template source, with merge tags, and putting it in an HTML page using the <script> tag
  2. Getting the source from your html page, and, compiling it into a Handlebars template
  3. Using Handlebars to create the HTML for one data object
  4. Looping over an array of objects to create the HTML for each item in the array

Sending data to the template

When sending data to a template system, there is a unique, text-based identifier that corresponds to the data source for each piece of data.

If you've ever worked with a mailing list newsletter, like a mail merge in Microsoft Office or an email campaign in MailChimp or Constant Contact, then you might be familiar with a concept called merge tags or merge fields. These are the labels that identify each piece of data, give it a name, and a location for where that data goes in the template.

Some examples of merge field names are: first_name, membership_id, or postal_code.

An example of an email template might start like this:

Date: [date]
Member ID: [membership_id]

Applicant:
[first_name] [last_name]
[company]
[street]
[city], [state] [postal_code]

Dear [first_name],
    We would like to congratulate you on...

Merge variables in Handlebars

Every template language has come up with some clever syntax for putting the merge tag into the template. Here are some examples from other systems.

MailChimp: uses a star and a pipe around the merge tag.


<p>Hi *|FIRST_NAME|*,</p>
<p><a href="*|PASSWORD_RESET_URL|*">Reset your password</a> using this link.</p>


MS Word: uses a double arrows.


Dear <<FirstName,>>
  In reference to case #<<FilingNumber>> we are writing to request the following additional...

In Handlebars, the double curly brace syntax is used around a merge tag variable.


<div>
  <h2>Patent status</h2>
  <p>Filing date: {{filingDate}}</p>
  <p>Grant date: {{publicationDate}}</p>
</div>

Installation

Handlebars a JavaScript library, so you link to it your html page the same as any other script. Make sure it's included before any of your own JS files.

<script src="js/handlebars.js"></script>
<script src="js/myApp.js"></script>

Like most popular open source JavaScript libraries, it is hosted for free for you on a CDN. You can either include a copy of the libary in your project and link to it locally (above) or link to the remote copy on the CDN (below).

<script src="https://cdnjs.cloudflare.com/ajax/libs/handlebars.js/4.0.5/handlebars.js"></script>
<script src="js/myApp.js"></script>

Handlebars flow

  1. Write the source HTML Template (with merge variables) using a script tag
  2. Compile the template into a function using Handlebars.compile. Compile returns a function.
  3. Call this new function and pass a JSON object as the argument. The function returns the final HTML.
  4. Insert this new final html into a DOM element.
